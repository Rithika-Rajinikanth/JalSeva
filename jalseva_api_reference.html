💧 JalSeva — API Sample Credentials

Copy-paste ready credentials and request bodies for every user role and every API phase. Covers Register → Login → All endpoints per role.

How to use: (1) Register the user → (2) Login via /api/v1/auth/token → (3) In Swagger UI click the 🔒 Authorize button, enter username + password → (4) Execute any endpoint with the token auto-attached. Replace PASTE-ALERT-ID-HERE with the id UUID from the alert creation response.
🏘️
Panchayat User PANCHAYAT
Village-level reporter — submits water scarcity alerts on behalf of the community
Scenario: Sarpanch Lakshmi Devi, Kothapalli Village, Warangal — 450 households have no water for 5 days after two borewells went dry. She registers, logs in, submits the alert, then tracks the timeline.
1
Register
POST
POST /api/v1/auth/register
Request Body — JSON
{
  "email":        "lakshmi@kothapalli.gov.in",
  "password":     "password123",
  "full_name":    "Sarpanch Lakshmi Devi",
  "role":         "PANCHAYAT",
  "organization": "Kothapalli Gram Panchayat",
  "panchayat":    "Kothapalli",
  "district":     "Warangal",
  "state":        "Telangana",
  "phone":        "+91-9876543210"
}
✓Expected: HTTP 201. Save the id field (UUID) — it is your user ID.
2
Authorize / Login
POST
POST /api/v1/auth/token Send as form-data, not JSON
Form-Data Fields
username : lakshmi@kothapalli.gov.in
password : password123
⚠In Swagger UI: click 🔒 Authorize at the top of the page. Enter username and password. Click Authorize. The token is now attached to all requests automatically.

Independent Field Validator VALIDATOR
The ONLY role that can close an alert — must submit physical field verification with evidence
Scenario: Priya Sharma, WaterAid India — visits Kothapalli on Feb 12. Interviews households, inspects new borewell, reviews quality certificate, submits APPROVED validation. Her decision permanently closes the alert.
🔒Critical Rule: An alert can ONLY reach CLOSED status through a VALIDATOR submitting decision: "APPROVED". No officer, admin, or system action can bypass this gate.

POST
POST/api/v1/alerts/{alert_id}/validateAlert must be in PENDING_VERIFICATION

  "validator_name":    "Priya Sharma",
  "validator_org":     "WaterAid India - Telangana Chapter",
  "method":            "Field visit Feb 12. Interviewed 15 households across 3 habitations. Inspected new borewell. Reviewed water quality test certificate.",
  "findings":          "New borewell operational since Feb 10. Water quality meets BIS 10500 (pH 7.2, TDS 320mg/L, zero bacterial contamination). All 450 households confirm supply restored.",
  "evidence_reviewed": "Borewell photos (7), water quality certificate TSPCB-2024-0219, contractor completion cert, 15 signed household feedback forms, GPS 17.9284 79.8945",
  "community_feedback":"Adequate response. Women note improved time savings on water collection. No disputes raised.",
  "recommendations":   "Monitor groundwater monthly April-June. Consider 2 rainwater harvesting structures.",
  "decision":          "APPROVED"
}
✓APPROVED → Alert status permanently set to CLOSED. closed_at timestamp sealed. Gemini scores coherence of findings vs resolution claim.

  "evidence_reviewed": "Site GPS-tagged photographs, 8 household interview transcripts",

  "recommendations":   "Immediate re-investigation required. Contractor accountability audit recommended.",
  "decision":          "DISPUTED"
}
⚠DISPUTED → Alert status = RESOLUTION_DISPUTED. Alert re-enters active lifecycle. SLA enforcement resumes from the top.
